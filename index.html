📚 GitHub Pages Hosting Guide

Deploy this app as a live website — free, permanent, shareable link.

🌐
Your app will be live at
https://yourusername.github.io/kevin-medicare
1
Create a GitHub Account
→ Go to github.com
→ Click Sign up (top right)
→ Enter email, password, username
→ Verify your email
✅ Free forever, no credit card needed
2
Create a New Repository
→ Click the + icon (top right of GitHub) → New repository
→ Repository name: kevin-medicare
→ Visibility: select Public ← very important!
→ Scroll down → click Create repository
Leave all other settings as default
3
⚠️ Rename & Upload the File
→ First, rename the downloaded file to exactly index.html
⚠️ This is the #1 cause of blank pages — file MUST be named index.html

→ In your new repo, click Add file → Upload files
→ Drag & drop your index.html file
→ Scroll down → click Commit changes (green button)
4
Enable GitHub Pages
→ In your repo, click Settings tab (top navigation)
→ In the left sidebar scroll down to → click Pages
→ Under Source select: Deploy from a branch
→ Branch dropdown: select main
→ Folder dropdown: select / (root)
→ Click Save
5
Wait 2–3 Minutes & Go Live!
→ GitHub builds your site automatically
→ Stay on the Pages settings page and refresh after 2 mins
→ A green banner will appear: ✅ Your site is live at https://...
→ Click that link — your app is now public!
🎉 Share the link with anyone — works on mobile too
6
First Time Opening? Add Your Gemini Key
→ When you (or anyone) opens the live URL for the first time, the Setup Wizard appears automatically
→ Get a free key at aistudio.google.com/app/apikey
→ Paste key → Save → App works instantly!
Key is saved in your browser — you only do this once per device
🛠 Why Blank Pages Happen — All Fixed in This Version
✅File not named index.html — causes 404. Always rename before uploading.
✅Opening as local file (file://) — API calls blocked. This app detects it and shows clear instructions.
✅No API key — app used to crash silently. Now shows setup wizard automatically.
✅JS error crashes page — all errors are caught and displayed as helpful messages.
✅Repo set to Private — GitHub Pages doesn't work on private repos (free plan). Must be Public.
